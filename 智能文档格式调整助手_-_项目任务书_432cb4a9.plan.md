# 智能文档格式调整助手 - 项目任务书

## 一、项目概述

### 1.1 项目背景

开发一个智能文档格式调整助手，通过截屏分析用户需求，调用大模型智能诊断和修复Word文档格式问题，提升文档编辑效率。

### 1.3 技术选型

- **UI框架**: CustomTkinter 5.x (Python原生GUI)
- **图像处理**: Pillow (截图显示与处理)
- **Word集成**: pywin32 (Windows COM接口)
- **大模型**: OpenAI GPT-4/3.5, Anthropic Claude, 本地Ollama
- **部署模式**: 单体本地部署（无需网络，隐私友好）
- **Python版本**: Python 3.8+ (推荐 3.10-3.12)

**架构说明**: CustomTkinter 提供原生 Python GUI，UI 层直接调用 core 模块的函数，无需额外的 API 桥接层。

### 2.2 核心模块说明

#### UI层 (`ui/` - HTML/CSS/JS)
- **index.html**: 主页面结构，模块化布局
- **styles.css**: 样式表，支持深色/浅色主题
- **app.js**: 前端主逻辑，处理用户交互
- **components/**: 可复用UI组件（截屏预览、对话面板等）
- **优势**: 前端生态成熟，开发灵活，可快速迭代

#### API桥接层 (`api.py`)
- **作用**: 连接 HTML 前端与 Python 后端
- **实现**: PyWebView 提供的 `window.pywebview.api`
- **方式**: Python 暴露方法，前端 JS 直接调用

```python
# api.py 示例
class Api:
    def capture_screen(self):
        """截屏"""
        return screenshot_manager.capture()

    def analyze_image(self, image_data: str):
        """分析图片"""
        return llm_service.analyze(image_data)

    def fix_format(self, plan: dict):
        """执行格式修复"""
        return format_fixer.execute(plan)
```

#### 业务逻辑层 (`core/`)
- **截屏管理**: 区域截屏、OCR识别、格式预分析
- **Word连接器**: 通过COM接口操作Word文档
- **格式分析器**: 检测文档格式问题
- **格式修复器**: 执行大模型返回的修复指令
- **LLM服务**: 多模型调用 (OpenAI/Claude/Ollama)

#### Skills层 (`skills/`)
- 字体样式、段落格式、页面布局等可组合的操作单元

#### 配置层 (`config/`)
- 大模型API配置
- 应用参数设置
- 会话历史存储

### 2.3 数据流向

```
用户操作(HTML界面)
       │
       ▼
前端JS (app.js)
       │ fetch/window.pywebview.api
       ▼
Python API (api.py)
       │
       ├──► core/screenshot_manager.py ──► Word COM
       │
       ├──► core/llm_service.py ──► OpenAI/Claude/Ollama
       │
       └──► core/format_fixer.py ──► Word COM
```


## 三、核心功能详述

> 注：采用 CustomTkinter 实现 UI 层，Python 直接调用 core 模块。

### 3.1 智能截屏模块

- **区域选择**: 自由框选、窗口吸附、全屏截屏
- **OCR识别**: 提取截屏中的文本内容（调用本地OCR或LLM视觉能力）

- **预览对比**: 修复前后效果对比

### 5.2 Python API 接口示例

```python
# api.py
import webview
from core import screenshot_manager, llm_service, format_fixer

class Api:
    """前端可调用的Python API"""

    def capture_screen(self):
        """截屏并返回base64图片"""
        return screenshot_manager.capture()

    def analyze_image(self, image_data: str, request: str):
        """分析图片并返回修复建议"""
        return llm_service.analyze(image_data, request)

    def fix_format(self, plan: dict):
        """执行格式修复计划"""
        return format_fixer.execute(plan)

    def get_models(self):
        """获取可用大模型列表"""
        return llm_service.get_available_models()

    def set_model(self, provider: str, model: str):
        """切换大模型"""
        llm_service.set_provider(provider, model)

    def undo_last(self):
        """撤销上一步操作"""
        return format_fixer.undo()
```

### 5.3 前端调用示例

```javascript
// ui/app.js

// 初始化时获取模型列表
async function init() {
    const models = await window.pywebview.api.get_models();
    renderModelSelector(models);
}

// 截屏
async function capture() {
    const imageData = await window.pywebview.api.capture_screen();
    document.getElementById('preview').src = `data:image/png;base64,${imageData}`;
}

// 分析图片
async function analyze(request) {
    const imageData = getCurrentImage();
    const result = await window.pywebview.api.analyze_image(imageData, request);
    renderAnalysisResult(result);
}

// 执行修复
async function executeFix(plan) {
    const success = await window.pywebview.api.fix_format(plan);
    if (success) {
        showNotification('格式修复成功！');
    }
}
```

## 八、风险与对策

| 风险项 | 风险等级 | 对策 |
|--------|---------|------|
| Word COM不稳定 | 中 | 增加重试机制、备用python-docx方案 |
| 大模型幻觉 | 高 | 严格解析验证、操作预览确认 |
| 跨平台兼容性 | 中 | 优先Windows，Mac/Linux使用python-docx替代COM |
| 性能瓶颈 | 低 | 异步处理、截屏压缩优化（本地调用无网络延迟） |
| LLM API不可用 | 中 | 降级到本地Ollama，显示友好提示 |

**额外优势**（相比分离架构）:

- 无网络通信失败风险
- 无服务端部署复杂度
- 截图数据完全本地处理，隐私更安全
