# 智能文档格式调整助手 - 项目任务书

## 一、项目概述

### 1.1 项目背景

开发一个智能文档格式调整助手，通过截屏分析用户需求，调用大模型智能诊断和修复Word文档格式问题，提升文档编辑效率。

### 1.2 系统架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        系统架构图                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌─────────────┐                              ┌─────────────┐ │
│  │   客户端    │                              │   服务端    │ │
│  │  (PyQt6)    │ ───── HTTPS/REST API ─────→ │  (Flask)    │ │
│  └─────────────┘                              └─────────────┘ │
│         │                                            │         │
│         │                                            │         │
│  ┌──────┴──────┐                            ┌───────┴───────┐ │
│  │ Word (COM)  │                            │   大模型池    │ │
│  │ 本地操作     │                            │ OpenAI/Claude │ │
│  └─────────────┘                            └───────────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 技术选型

- **服务端**: Flask + Python 3.10+
- **客户端**: PyQt6 6.6+
- **Word集成**: pywin32 (COM接口)
- **大模型**: OpenAI GPT-4/3.5, Anthropic Claude, 本地Ollama
- **部署模式**: 混合部署（客户端本地，服务端可本地/云端）

## 三、核心功能详述

### 3.1 智能截屏模块

- **区域选择**: 自由框选、窗口吸附、全屏截屏
- **OCR识别**: 提取截屏中的文本内容
- **格式预分析**: 识别文档类型、当前格式状态
- **多截图管理**: 支持多张截图对比分析

### 3.2 大模型交互模块

- **多模型切换**: OpenAI/Claude/本地模型热切换
- **上下文管理**: 维护对话历史和文档状态
- **提示词优化**: 针对格式调整优化的提示词模板
- **流式响应**: 支持大模型流式输出

### 3.3 格式分析与修复模块

- **格式问题检测**: 间距、字体、对齐、页边距等
- **一键修复**: 执行大模型返回的修复指令
- **预览对比**: 修复前后效果对比
- **撤销支持**: 支持撤销修复操作

### 3.4 Skills库

| Skill分类 | 功能描述 | 示例 |

|-----------|---------|------|

| 字体样式 | 调整字体、大小、颜色 | "将标题改为微软雅黑24号加粗" |

| 段落格式 | 行距、缩进、对齐 | "设置段落行距为1.5倍" |

| 页面布局 | 页边距、纸张方向 | "设置A4纵向，上下边距2.5cm" |

| 样式应用 | 应用预设样式 | "应用标题1、标题2样式" |

| 目录生成 | 自动生成目录 | "根据标题自动生成目录" |

| 页眉页脚 | 设置页眉页脚 | "添加页码和公司名称" |

## 五、API接口设计

### 5.1 接口列表

| 方法 | 路径 | 功能 | 请求格式 |

|------|------|------|----------|

| POST | `/api/v1/analyze` | 分析截屏并返回修复建议 | multipart/form-data |

| POST | `/api/v1/execute` | 执行格式调整指令 | application/json |

| GET | `/api/v1/skills` | 获取可用Skills列表 | - |

| POST | `/api/v1/chat` | 连续对话交互 | application/json |

| GET | `/api/v1/models` | 获取可用大模型列表 | - |

### 5.2 请求示例

```json
POST /api/v1/analyze
{
  "screenshot": "base64编码图片",
  "user_request": "将标题字体改为微软雅黑24号",
  "document_type": "word",
  "history_enabled": true
}
```

### 5.3 响应示例

```json
{
  "success": true,
  "analysis": {
    "detected_issues": ["标题字体为宋体", "字号不统一"],
    "suggested_actions": [
      {
        "skill": "set_font",
        "params": {"font_name": "微软雅黑", "size": 24, "target": "title"}
      }
    ]
  },
  "execution_plan": {
    "steps": ["选中标题段落", "设置字体", "设置字号"],
    "estimated_time": "2秒"
  }
}
```
